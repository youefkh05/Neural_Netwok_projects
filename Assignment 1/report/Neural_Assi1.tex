\section{Part 2: some classical ML modeling}

Part 2 uses the recovered ground-truth labels from \texttt{pipeline3/full\_labels\_vector.csv} and evaluates three feature representations: DCT, PCA, and HOG. Each feature set was tested with K-means per class using 1, 4, 16, and 32 clusters per class, and with SVM using linear and RBF kernels.

\begin{itemize}
    \item \textbf{Best overall classifier:} HOG + RBF SVM with 99.80\% accuracy.
    \item \textbf{Best K-means classifier:} HOG + 16 clusters per class with 99.40\% accuracy.
    \item \textbf{PCA dimensionality:} 161 components were required to reach at least 95\% explained variance.
\end{itemize}

\section{Pipeline 1: K-Means Bootstrapping with Active SVM Refinement}

\begin{figure}[H]
\centering
\includegraphics[width=0.95\textwidth]{Pipeline1_Flowchart.png}
\caption{Pipeline 1: K-Means + SVM + Active Learning}
\end{figure}

The flowchart highlights the main design idea of Pipeline 1: use low-cost cluster-level supervision first, then spend manual effort only on uncertain boundary samples where human labels give maximum gain.

The pipeline consists of the following steps:
\begin{itemize}
    \item Load dataset (10,000 BMP images)
    \item Extract HOG features
    \item Apply K-Means clustering
    \item Label clusters (bootstrapping)
    \item Train SVM classifier
    \item Apply active learning for refinement
\end{itemize}

% =========================
% STEP 1
% =========================
\subsection{Step 1 - Feature Extraction and K-Means Clustering}

Histogram of Oriented Gradients (HOG) was used to extract features from images.

\begin{itemize}
    \item Image size: 32 x 32
    \item Feature vector size: 1764
\end{itemize}

\begin{figure}[H]
\centering
\includegraphics[width=0.7\textwidth]{HOG_Feature.png}
\caption{Example of HOG Feature Representation}
\end{figure}

The HOG visualization illustrates why this feature performs well in our experiments: it preserves local edge orientation patterns that are stable across many writing styles and minor geometric variations.

% =========================
% STEP 2
% =========================
\subsection{Step 2 - Human Labelling of Clusters (Bootstrapping Phase)}

K-Means clustering was applied with different values of K:

\begin{itemize}
    \item K = 40
    \item K = 60
    \item K = 80
\end{itemize}

\begin{figure}[H]
\centering
\includegraphics[width=0.7\textwidth]{k_vs_accuracy.png}
\caption{K vs Accuracy Before SVM}
\end{figure}

This plot shows that increasing K improves the initial pseudo-label quality up to the tested range, because smaller and more coherent clusters reduce mixed-class assignments during cluster bootstrapping.

% =========================
% STEP 3 AND STEP 4
% =========================
\subsection{Step 3 - Initial SVM Training and Step 4 - Active Refinement: Labelling Boundary Images}

A multi-class SVM with RBF kernel (One-vs-One strategy) was used.

Active learning was applied as follows:
\begin{itemize}
    \item Select 30 least confident samples (lowest margin)
    \item Relabel manually
    \item Assign weight = 100
    \item Retrain SVM
\end{itemize}

The process stops when:
\begin{itemize}
    \item Accuracy $\geq 99\%$
    \item Improvement $< 0.1\%$
\end{itemize}

\begin{figure}[H]
\centering
\includegraphics[width=0.7\textwidth]{svm_bar_chart.png}
\caption{Accuracy Before vs After SVM}
\end{figure}

The bar chart quantifies the gain from SVM refinement over raw cluster labels. The gain is larger when initial clusters are weaker, and smaller when clustering is already highly pure.

\section{Pipeline 2: Manual Seed + Augmentation + Self-Training}

This pipeline adopts a semi-supervised learning approach starting from a small, high-quality human-labelled seed set. The model progressively expands its knowledge using data augmentation, active learning, and self-training.

% FLOWCHART (TIKZ)
% =========================
\subsection{Pipeline Diagram}

\begin{figure}[H]
\centering

The Pipeline 2 diagram emphasizes a different strategy: begin from a trusted manual seed, then expand supervision through augmentation, confidence-based pseudo-labeling, and iterative hard-sample correction.

% =========================
% CONCLUSION
% =========================
\subsection{Discussion}

Pipeline 2 demonstrates the effectiveness of semi-supervised learning. Starting from a small, high-quality seed set, the model successfully expands its training data using both active learning and self-training.

Although the final accuracy (99.04\%) is slightly below Pipeline 1, the reduction in manual labeling effort is significant. This pipeline better reflects real-world scenarios where labeled data is scarce and expensive.

The combination of:
\begin{itemize}
    \item Human supervision (high confidence)
    \item Model confidence (self-training)
    \item Iterative refinement
\end{itemize}

creates a scalable and efficient labeling strategy.
