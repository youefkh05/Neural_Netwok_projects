\paragraph{1. Scatter Plot \& Outlier Identification}

A scatter plot of $y$ (number of insured persons) versus $x$ (years since 1987) reveals
a generally decreasing trend from $\approx 12{,}400$ down to $\approx 7{,}200$.
However, one data point stands out sharply:
\begin{itemize}
  \item $x = 3$ (year 1990): $y = 1050$
\end{itemize}

This value is dramatically lower than the surrounding data, which are all within
approximately $7{,}200$ to $12{,}400$. The sharp drop breaks the otherwise smooth
downward trend. This observation is therefore identified as a clear outlier, most
likely due to a data entry error (e.g.\ $10{,}500$ instead of $1{,}050$).

\paragraph{2. Regression Models (WITH Outlier)}

We apply both OLS and matrix-form methods to compute linear model parameters,
then use the matrix form for quadratic and cubic models.

\subparagraph{2.1 Linear Model}

\textbf{Hand Analysis (OLS Method)}

The equation for the linear model is:
\begin{equation}
  \hat{y}_i = \beta_0 + \beta_1 x_i
\end{equation}

The key formulas:
\begin{equation}
  \hat{\beta}_1 = \frac{\displaystyle\sum_{i=1}^{n} x_i y_i - \frac{\bigl(\sum x_i\bigr)\bigl(\sum y_i\bigr)}{n}}
                       {\displaystyle\sum_{i=1}^{n} x_i^2 - \frac{\bigl(\sum x_i\bigr)^2}{n}},
  \qquad
  \hat{\beta}_0 = \bar{y} - \hat{\beta}_1\,\bar{x}
\end{equation}

$n = 10$,\quad $\bar{y} = 83350/10 = 8335$,\quad $\bar{x} = 45/10 = 4.5$

\[

\]

\[
\hat{\beta}_0 = 8335 + 367.57 \times 4.5 = 9989.09
\]

\textbf{Hand Analysis (Matrix Form)}

\[
\bm{\beta} = (X^T X)^{-1} X^T Y
\]

\[
\bm{\beta} = \begin{pmatrix}\beta_0\\\beta_1\end{pmatrix},\quad
X = \begin{pmatrix}1&0\\1&1\\1&2\\1&3\\1&4\\1&5\\1&6\\1&7\\1&8\\1&9\end{pmatrix},\quad
Y = \begin{pmatrix}12400\\10900\\10000\\1050\\9500\\8900\\8000\\7800\\7600\\7200\end{pmatrix}
\]

\[
\begin{pmatrix}\beta_0\\\beta_1\end{pmatrix} = \begin{pmatrix}9989.09\\-367.576\end{pmatrix}
\]

Linear equation:
\begin{equation}
  \hat{y}_i = 9989.09 - 367.576\,x
\end{equation}

\textbf{R² Verification Table}

\[
\bar{y} = \frac{\sum y_i}{n} = \frac{83350}{10} = 8335
\]

\section{Pipeline 1: K-Means Bootstrapping with Active SVM Refinement}

\caption{Pipeline 1: K-Means + SVM + Active Learning}
\end{figure}

The flowchart highlights the main design idea of Pipeline 1: use low-cost cluster-level supervision first, then spend manual effort only on uncertain boundary samples where human labels give maximum gain.

The pipeline consists of the following steps:
\begin{itemize}
    \item Load dataset (10,000 BMP images)
    \item Extract HOG features
    \item Apply K-Means clustering
    \item Label clusters (bootstrapping)
    \item Train SVM classifier
    \item Apply active learning for refinement
\end{itemize}

% =========================
\subsection{Step 1 - Feature Extraction and K-Means Clustering}

Histogram of Oriented Gradients (HOG) was used to extract features from images.

\begin{itemize}
    \item Image size: 32 x 32
    \item Feature vector size: 1764
\end{itemize}

\begin{figure}[H]

% =========================

% =========================
\subsection{Step 2 - Human Labelling of Clusters (Bootstrapping Phase)}

K-Means clustering was applied with different values of K:

\begin{itemize}
    \item K = 40
    \item K = 60
    \item K = 80
\end{itemize}

\begin{figure}[H]
\centering

\caption{K vs Accuracy Before SVM}
\end{figure}

This plot shows that increasing K improves the initial pseudo-label quality up to the tested range, because smaller and more coherent clusters reduce mixed-class assignments during cluster bootstrapping.

% =========================
% STEP 3 AND STEP 4
% =========================
\subsection{Step 3 - Initial SVM Training and Step 4 - Active Refinement: Labelling Boundary Images}

A multi-class SVM with RBF kernel (One-vs-One strategy) was used.

Active learning was applied as follows:
\begin{itemize}
    \item Select 30 least confident samples (lowest margin)
    \item Relabel manually
    \item Assign weight = 100
    \item Retrain SVM
\end{itemize}

The process stops when:
\begin{itemize}
    \item Accuracy $\geq 99\%$
    \item Improvement $< 0.1\%$
\end{itemize}

\caption{Accuracy Before vs After SVM}
\end{figure}

The bar chart quantifies the gain from SVM refinement over raw cluster labels. The gain is larger when initial clusters are weaker, and smaller when clustering is already highly pure.

\section{Pipeline 3: LLM-Based Automatic Labelling with Agreement Validation}

Pipeline 3 was implemented with OpenRouter-based multimodal models and evaluated on the practical 500-image matrix. During experimentation, several runs were completed successfully, while some long/full runs were only partially completed because of API-credit limits (HTTP 402) and provider payload instability on some requests.

The implemented setup included:
\begin{itemize}
    \item Balanced practical subset matrix (500 images total, 50 per class).
    \item Image resizing to 224$\times$224 before API submission.
    \item Optional batched requests (5 or 10 images per API call).
    \item Robust retry handling for timeout/429 and fallback for HTTP 400 batch failures.
    \item A labeled reference sheet (digits 0--9) sent as an additional context image.
\end{itemize}
