\section{Problem 6: GAN-based Synthetic Data Generation}

\subsection{Step 1: GAN Training and Initial Generation}

A Generative Adversarial Network (GAN) was trained using a reduced dataset of 350 real samples per digit. The goal was to generate synthetic handwritten digits resembling the MNIST dataset.

The GAN consisted of a convolutional discriminator and a transposed convolutional generator. The training process was conducted for several epochs until visually acceptable digit-like samples were obtained.

The generated images were slightly blurred but still recognizable, indicating that the GAN successfully captured the general structure of handwritten digits.

\begin{figure}[H]
    \centering
    \includegraphics[width=0.6\linewidth]{gan_generated_samples.png}
    \caption{Samples generated by the GAN after training}
\end{figure}

The total training time was recorded, along with the generation time of synthetic samples, to evaluate computational cost.

\subsection{Step 2: Effect of Synthetic Data on Classification Accuracy}

In this step, the generated synthetic data was combined with different amounts of real data (350, 750, and 1000 samples per digit). The classifier from Problem 5 (LeNet) was trained on these combined datasets.

Figure~\ref{fig:gan_results} shows the relationship between classification accuracy and the number of GAN-generated samples per digit.

\begin{figure}[H]
    \centering
    \includegraphics[width=0.8\linewidth]{results_plot.png}
    \caption{Accuracy vs. number of GAN-generated samples per digit for different real dataset sizes}
    \label{fig:gan_results}
\end{figure}

From the results, several observations can be made:

\begin{itemize}
    \item Adding a moderate number of synthetic samples (around 1000 per digit) improves the accuracy when limited real data is available.
    \item The best improvement is observed for the case of 350 real samples, where synthetic data significantly boosts performance.
    \item Increasing synthetic data beyond a certain point (e.g., 1500 or 2000 samples) does not improve performance and may slightly degrade it.
    \item This degradation is due to the limited quality and diversity of GAN-generated images, which can introduce noise and bias into the training process.
\end{itemize}

\subsection{Step 3: Reducing the Need for Real Data}

The objective of this step was to determine how much real data can be replaced by synthetic data without significantly affecting performance.

The baseline case using only real data (1000 samples per digit) achieved an accuracy of approximately $97\%$.

When using only 350 real samples per digit combined with 1000 GAN-generated samples, the classifier achieved an accuracy close to $96.7\%$, which is comparable to the full real-data scenario.

This demonstrates that GAN-generated data can effectively reduce the requirement for real labeled data while maintaining high classification accuracy.

However, it is important to note that excessive reliance on synthetic data leads to diminishing returns and may reduce performance due to artifacts in generated images.

\subsection{Discussion}

The results confirm that GANs can serve as a powerful tool for data augmentation, especially in scenarios where labeled data is limited.

Key takeaways include:

\begin{itemize}
    \item GAN-generated data can partially replace real data with minimal loss in accuracy.
    \item There exists an optimal balance between real and synthetic data.
    \item Overusing synthetic data can negatively affect model generalization.
\end{itemize}

Overall, the experiment demonstrates that synthetic data generation is a viable approach to reducing data collection requirements in machine learning systems.
