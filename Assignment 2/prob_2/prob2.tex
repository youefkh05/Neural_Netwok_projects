\section{CNN Experiments (LeNet-style)}

\subsection{Design Decisions}
We implemented a baseline \texttt{LeNet5} with the following structure:
\begin{itemize}
  \item Input image: $28\times28\times1$
  \item Convolutional layer 1: $5\times5$, 6 filters $\rightarrow$ $24\times24\times6$
  \item Average pooling $2\times2$ $\rightarrow$ $12\times12\times6$
  \item Convolutional layer 2: $5\times5$, 16 filters $\rightarrow$ $8\times8\times16$
  \item Average pooling $2\times2$ $\rightarrow$ $4\times4\times16$
  \item Flatten $\rightarrow$ 256 features
  \item Fully connected layers: 256 $\rightarrow$ 120 $\rightarrow$ 84 $\rightarrow$ 10 logits
\end{itemize}

\begin{figure}[ht]
  \centering
  \includegraphics[width=0.6\linewidth]{lenet5.png}
  \caption{LeNet-5 architecture (baseline).}
  \label{fig:lenet}
\end{figure}

Variants added to investigate capacity and regularisation effects:
\begin{description}
  \item[Wide:]
  The spatial flow becomes:
  \begin{itemize}
    \item Input image: $28\times28\times1$
    \item Convolutional layer 1: $5\times5$, \textbf{16} filters $\rightarrow$ $24\times24\times\textbf{16}$
    \item Average pooling $2\times2$ $\rightarrow$ $12\times12\times\textbf{16}$
    \item Convolutional layer 2: $5\times5$, \textbf{32} filters $\rightarrow$ $8\times8\times\textbf{32}$
    \item Average pooling $2\times2$ $\rightarrow$ $4\times4\times\textbf{32}$
    \item Flatten $\rightarrow$ \textbf{512} features
    \item Fully connected layers: \textbf{512} $\rightarrow$ 120 $\rightarrow$ 84 $\rightarrow$ 10 logits
  \end{itemize}
  \item[Deep:]
  The spatial flow becomes:
  \begin{itemize}
    \item Input image: $28\times28\times1$
    \item Convolutional layer 1: $5\times5$, 6 filters $\rightarrow$ $24\times24\times6$
    \item Average pooling $2\times2$ $\rightarrow$ $12\times12\times6$
    \item Convolutional layer 2: $5\times5$, 16 filters $\rightarrow$ $8\times8\times16$
    \item Convolutional layer 3: $3\times3$, \textbf{32} filters $\rightarrow$ $8\times8\times\textbf{32}$
    \item Average pooling $2\times2$ $\rightarrow$ $4\times4\times\textbf{32}$
    \item Flatten $\rightarrow$ \textbf{512} features
    \item Fully connected layers: \textbf{512} $\rightarrow$ 120 $\rightarrow$ 84 $\rightarrow$ 10 logits
  \end{itemize}
  \item[Sigmoid:]
  Same as the baseline in terms of spatial sizes and feature dimensions, but with Sigmoid activations instead of ReLU.
\end{description}

Each variant is constructed by a factory function and run with the same training pipeline to ensure fair comparison.

\subsection{Parameters of the Experiment}
\begin{itemize}
  \item Learning rate: 0.001.
  \item Batch size: 128.
  \item Epochs: 5.
  \item Metrics: train accuracy, test accuracy, training wall time, testing wall time.
\end{itemize}

\subsection{Discussion}
The feature-based classifiers show a consistent pattern: increasing model capacity usually helps, but the best trade-off depends on the feature representation. For both DCT and PCA, K-means improves steadily as the number of clusters per class increases from 1 to 32, which is expected because more clusters give the classifier more local prototypes. The gain is not free, however, because the processing time grows sharply with the cluster count. Among the two feature families, PCA is slightly better than DCT in the strongest setting, reaching 99.0\% with RBF-SVM versus 98.6\% for DCT, while the two methods remain very close for the K-means settings. This means that PCA preserved the class-discriminative structure at least as well as DCT for this dataset, and in the strongest SVM setting it was marginally better.

The AutoEncoder features are the weakest of the three in this setup. Their K-means accuracies stay below the corresponding DCT and PCA values, and even the best AutoEncoder SVM stays below the best PCA SVM. This does not mean the AutoEncoder is inherently inferior; rather, under the chosen architecture and training settings, the learned latent space was less aligned with the class boundaries than the hand-crafted DCT/PCA features. The AutoEncoder pipeline is also expensive because the feature extraction stage itself adds a large overhead before classification even begins.

The MLP results reinforce the same conclusion. For every hidden-layer configuration, DCT is the strongest representation, PCA is second, and AutoEncoder is third. The 1-hidden and 4-hidden variants are essentially tied on DCT at 97.0\%, while the 3-hidden model is slightly lower at 96.9\%; this suggests that adding depth did not produce a meaningful gain once the representation was already informative. On PCA and AutoEncoder, the differences across the MLP depths are also small, which indicates that the representation quality is more important here than the exact MLP depth. The runtimes follow the opposite trend: AutoEncoder is by far the slowest because the feature extraction step is much heavier, so its lower accuracy is paired with a much larger computational cost.

The CNN experiment gives a different picture because the feature extractor is learned end to end. The baseline LeNet-style model reaches 95.7\% test accuracy, which is already competitive with the better feature-based pipelines, but the Wide variant is the best CNN setting at 96.2\%. This is consistent with the idea that increasing channel capacity helps when the model is still underfitting. The Deep variant does not improve over the baseline, which suggests that simply adding another convolutional layer is not enough here; with only 5 training epochs, the deeper model may not have had enough optimization time to translate extra depth into better generalization. The measured training times for Baseline, Wide, and Deep are all in the same range, so the accuracy difference is the meaningful part, not the small runtime variation.

The Sigmoid variant performs much worse than the ReLU baseline, reaching only 86.2\% test accuracy. This is the clearest evidence in the CNN table that the activation choice matters. Sigmoid saturates for moderately large positive or negative inputs, which weakens gradient flow and makes optimization slower and less effective. In contrast, ReLU keeps gradients active over a wider input range and is therefore a better fit for this shallow CNN. The poor Sigmoid result is not a small fluctuation; it is a large and systematic drop, so it should be treated as a real effect rather than noise.

The timing numbers should be interpreted with care. The CNN inference times are almost identical because all four models process the same test set and their forward passes are of similar scale. Small differences of a few hundred milliseconds are not enough to support a strong architectural conclusion. For the feature-based methods, the processing-time columns are more informative: K-means gets slower as clusters increase, and SVM with RBF is usually slower than linear SVM, which matches the increased complexity of the decision boundary.

\subsection{Conclusion and Recommendations}
Overall, the results point to a clear ranking of the representations and models under the current experimental setup. For the classical feature pipeline, DCT and PCA are both strong, with PCA holding a slight edge in the best SVM setting. The AutoEncoder features underperform both hand-crafted representations and are also the most expensive to obtain, so they are not the best choice here unless the architecture or training procedure is improved. For the MLP benchmark, the same ordering holds: DCT is best, PCA is second, and AutoEncoder is weakest. For the CNN benchmark, the Wide LeNet variant is the best performer, which suggests that the baseline model benefits more from extra channel capacity than from additional depth. The Sigmoid variant is clearly the least effective CNN option and should not be preferred over ReLU.
